### <span style="color:orange">2026 年 1 月</span> — 1.21.4+ 版本采用 Item Display

- 玩家可能会发现，**Armor Stand 存在较明显的传送延迟问题**，包括实体在传送过程中渲染位置滞后于真实位置，以及由此带来的轨迹平滑效果。这会导致球的渲染位置和轨迹与实际模拟结果出现偏差。为解决这一问题，本数据包现已改用 **Item Display** 进行渲染。默认情况下，Item Display 不受传送延迟影响，因此可以 **精确还原球的位置与运动轨迹**，整体视觉效果与底层物理模拟高度一致。

- Item Display 的传送延迟行为可通过其 **`teleport_duration`** 数据进行控制。较大的取值会引入更明显的渲染延迟，同时使运动轨迹更加平滑。旧版基于 Armor Stand 的渲染效果大致相当于 `teleport_duration = 2`。  
  - `teleport_duration = 0`：完全离散的运动渲染  
  - `teleport_duration = 1`：在轨迹平滑与位置精度之间取得较好平衡（推荐）  
  - 更大的取值：可能会对轨迹产生过度平滑  
  具体配置方式请参考 **使用指南与演示**。

- 当 Item Display 发生旋转时，其 **`transformation`** 数据会持续更新，行为上类似于 Armor Stand 的 **`Pose`** 数据变化。尽管两者所采用的数学描述不同，但其均使用 **在世界坐标系中定义的三维旋转向量** 计算。

- *理论上可以将这个更新推送给更早的版本，但作者决定只支持 **1.21.4** 及更新的版本。鉴于早期版本中命令语法不断更改，为其重新适配将带来较高的维护成本，因此更倾向于保持旧版本的稳定状态，而只在新版本中引入这一改进。*

### <span style="color:orange">2025年九月</span> — AI开球模拟！

- 加入 **AI开球模拟**：用简易神经网络替代原有的弹性碰撞模型，开球效果更贴近现实。
- 数据生成与训练：在 Python 中用高精度物理仿真生成大量开球数据，训练神经网络学习球体压缩-反弹的瞬时过程；结合球堆对称性，模型可通过翻转和旋转泛化到全局。
- 实现方式：将神经网络参数手动转写为MC计分板运算，翻转/旋转也在游戏中完成，神经网络输出反向还原后给出每个球最终的速度和移动方向。
- 可在设置中开关此功能（默认开启），开启时八球开球会自动启用神经网络计算。提供四种模型复杂度，可自行权衡计算精度与性能。

### <span style="color:orange">2025年七月</span> — 中式八球已加入数据包，现在可以从大厅进入游戏，开始体验！

这是一个长期项目......

- 起初，设计的球体只是静态模型，只能直线移动，无旋转效果。当时只计划制作斯诺克和英式八球，用的是单色球。
- 随后，通过引入更多物理模拟，支持了母球控制和台球旋转相关的物理，球的运动表现更加真实。
- 接着，将旋转信息映射为盔甲架姿态，实现了可视化旋转效果，并引入编号球（仅限练习模式）。这项技术实现早于 Minecraft 1.20 的 Transformation Rendering 功能推出。
- 现在，中式八球规则已完成，实现了带编号球的单人和多人完整玩法。
